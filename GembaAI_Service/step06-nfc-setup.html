Step 6 / 9 — NFCタグ準備・登録

NFCタグ発行・TagMaster登録・現場との紐付け
🏠 TOP > Step6 NFCタグ準備

🎯 このステップのゴール

各現場で使用する物理NFCタグを準備し、Azure SQL の TagMaster と OnsiteTagRelation に登録します。1現場あたり最大6種類のタグが必要です。

📦 必要なNFCタグ数

1現場あたり6機能 × 現場数 = 必要タグ枚数

#機能mode値用途
1新規入場new-entry初回入場時の登録
2新規教育完了tech新規入場者教育受講記録
3通常入場entry毎日の出社打刻
4退場exit毎日の退社打刻
5KY活動kyactivityKY活動実施記録
6安全会議参加safety-reg安全会議参加署名代替
💡 タグの選定
推奨はNTAG424 DNA（暗号化機能付き、リプレイ攻撃対策あり）。cmac と piccdata の動的生成に対応している必要があります。

🛠 NFCタグ書き込み

1. タグ発行

  1. NTAG424タグを必要数調達
  2. 専用書き込みツール（NXP TagWriterなど）でURLを書き込み
  3. 書き込むURL形式: 
    https://face.gemba-ai.cloud/nfc-entry?cmac={動的}&piccdata={動的}&mode={機能}&ClientID={ClientID}

注意: cmac と piccdata はNTAG424の機能でタッチごとに動的生成されます。タグ自体にはテンプレートURLを書き込み、動的部分はチップが自動付与します。

2. タグ視認用ID（DisplayID）の決定

物理タグに人間が読める番号を貼り付けます（例: KY001, EN042 など）。台帳管理用です。

OnsiteTagRelation テーブル

各タグを現場と紐付けします。

INSERT INTO dbo.OnsiteTagRelation

VALUES

  (2, 5, 'A1B2C3D4E5F60718', 1, GETDATE()),  -- 通常入場
  (2, 5, 'B2C3D4E5F6071829', 1, GETDATE()),  -- 退場

OnsiteID は SPO の現場マスタの ID と一致させます。

🔍 登録確認

  1. テストスマホでNFCタグをタッチ
  2. face.gemba-ai.cloud のNFC打刻画面が表示されれば成功
  3. 顔認証 → 個人情報未登録ならエラー、登録済みなら次画面へ
⚠️ HTTPS必須
NFCタグからの遷移はHTTPSのみ。スマホブラウザでもセキュアコンテキストが必要です。

✅ 完了チェックリスト

  • 必要数のNTAG424タグを調達した
  • 各タグにテンプレートURLを書き込んだ
  • TagMasterに全タグを登録した
  • OnsiteTagRelationで現場×タグを紐付けた
  • テストスマホで実際にタッチして応答確認した
  • 視認用ID（DisplayID）が物理タグに貼られている
