Step 6 / 9 — NFCタグ準備・現場紐付け

物理タグの調達 ＋ 現場NFCタグ紐付け画面での登録
🏠 TOP > Step6 NFCタグ準備

🎯 このステップのゴール

各現場で使用する物理NFCタグを準備し、face.gemba-ai.cloud の管理画面から手動で現場と紐付けます。1現場あたり最大6種類のタグが必要です。

💡 SQL直叩きは不要
DBへの登録は管理画面が裏でやってくれます。クラウドパワー社の運用担当者は face.gemba-ai.cloud/onsitenfctag 画面でGUI操作するだけでOK。

📦 必要なNFCタグ数

1現場あたり最大6機能 × 現場数 = 必要タグ枚数

#機能タグ種別 (TagType)用途
1新規入場new-entry初回入場時の登録
2新規教育完了tech新規入場者教育受講記録
3通常入場entry毎日の出社打刻
4退場exit毎日の退社打刻
5KY活動kyactivityKY活動実施記録
6安全会議参加safety-reg安全会議参加署名代替
💡 タグの選定
推奨はNTAG424 DNA（暗号化機能付き、リプレイ攻撃対策あり）。cmac と piccdata の動的生成に対応している必要があります。書き込むテンプレートURLは: 
https://face.gemba-ai.cloud/nfc-entry?cmac={動的}&piccdata={動的}&mode={機能}&ClientID={ClientID}

🛠 NFCタグの物理準備

  1. NTAG424タグを必要数調達（現場数 × 6種類）
  2. 専用書き込みツール（NXP TagWriter等）でテンプレートURLを書き込み
  3. 物理タグに人間が読める掲示用IDのラベルを貼る
    • 命名規則例: NGK-0001（現場略称＋連番）
    • 奇数番=入場系、偶数番=KY系 など、運用に合わせて
  4. タグの物理ID（14桁のHEX）を控える（後で管理画面に入力する）

🌐 現場NFCタグ紐付け画面で登録

アクセス先

項目値
URLhttps://face.gemba-ai.cloud/onsitenfctag
画面名現場NFCタグ紐付け一覧
左サイドメニュー「現場NFCタグ紐付け」

登録手順

  1. 管理画面にM365 SSOでログイン
  2. 左メニュー「現場NFCタグ紐付け」を選択
  3. 画面上部のドロップダウンで対象の現場を選択（例: (仮称)ささしまライブマンション新築工事）
  4. 「+ 新しいアイテムを追加」ボタンをクリック
  5. 以下の項目を入力

入力項目

項目説明例
タグIDNTAG424の物理ID（14桁HEX）04501a927e7580
現場名称選択中の現場（自動入力）(仮称)ささしまライブマンション新築工事
タグ名機能識別の名前new-entry
タグ種別6種類から選択new-entry / entry / exit / kyactivity / tech / safety-reg
掲示用ID物理ラベルと同じIDNGK-0001
  1. 「保存」をクリック
  2. 1現場あたり最大6種類分（6機能）を繰り返し登録
💡 永賢組の登録例（参考）
ささしまライブマンション現場では、6種類のタグが NGK-0001 〜 NGK-0011（奇数番）で登録されています。
  • NGK-0001 → new-entry
  • NGK-0003 → entry
  • NGK-0005 → tech
  • NGK-0007 → exit
  • NGK-0009 → kyactivity
  • NGK-0011 → safety-reg

🔍 登録確認

  1. 登録した一覧画面に6行が表示されることを確認
  2. テストスマホで実際にNFCタグをタッチ
  3. face.gemba-ai.cloud/nfc-entry?... に遷移し、対応する画面（KY活動・新規入場・通常入場など）が表示されれば成功
  4. 顔認証 → 個人情報未登録ならエラー、登録済みなら次画面へ
⚠️ HTTPS必須
NFCタグからの遷移はHTTPSのみ。スマホブラウザでもセキュアコンテキストが必要です。

🔧 内部仕様（参考）

管理画面で登録すると、以下の2つのテーブルにレコードが作成されます（運用担当者は意識不要）。

  • TagMaster (Azure SQL): タグID/契約先ID/タグ名/タグ種別/掲示用ID
  • OnsiteTagRelation (SPO List): 契約先ID/現場ID/タグID

また、TagTouchLog テーブルでタッチ回数（Counter）を管理し、NTAG424のリプレイ攻撃対策を行っています。

✅ 完了チェックリスト

  • 必要数のNTAG424タグを調達した
  • 各タグにテンプレートURLを書き込んだ
  • 物理タグに掲示用IDラベルを貼った
  • face.gemba-ai.cloud/onsitenfctag で各現場×6機能のレコードを登録した
  • テストスマホで実際にタッチして応答確認した
  • 6種類すべての機能（新規入場/教育/入場/退場/KY/安全会議）の遷移を確認した
