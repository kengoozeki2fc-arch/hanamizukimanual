🎯 このステップのゴール

Step4で作成した8リストに、運用に必要な初期データを投入します。投入順序が重要です（参照関係があるため）。

📊 投入順序

[1] 社員資格マスタ ← 参照されるマスタを先に [2] 社員マスタ ← 社員資格IDを参照 [3] 危険作業マスタ ← KY活動の選択肢 [4] 危険回避策マスタ ← 危険作業IDを参照 [5] 協力業者マスタ ← 業者情報 [6] 現場マスタ ← 現場情報 [7] 現場協力業者紐付け ← 現場ID×業者IDで施工体制構築 [8] 社員実績 ← 社員ID×現場IDで現場履歴

🛠 各リストの投入手順

1. 社員資格マスタ

テナントが扱う資格名称を一覧で投入します。

・一級土木施工管理技士

2. 社員マスタ

テナントの社員情報を投入します。権限ロールの設定が重要です。

大関さん本人を「システム管理者」として登録するパターンが標準です。

3-4. 危険作業マスタ・危険回避策マスタ

6. 現場マスタ

初期登録対象の現場を投入します。現場名称・所長名・工期・略称ローマ字は必須です。

例:
現場名称: (仮称)○○マンション新築工事
所長名: ○○ ○○
工期(自): 2026-05-01
工期(至): 2027-12-31
略称ローマ字: MARUMARU

7. 現場協力業者紐付け

各現場の施工体制ツリーを構築します。1次請から順に登録していきます。

例: 現場「(仮称)○○マンション」
─ 1次: 株式会社サンエイ（階層1、上位なし）
　└ 2次: 東昌基設備株式会社（階層2、上位=サンエイ）
　　└ 3次: 株式会社□□（階層3、上位=東昌基設備）

8. 社員実績

各社員の現場担当履歴を投入します。新規企業の場合、現時点で関わっている現場のみで構いません。

🔄 一括投入のテクニック

方法A: SharePointのグリッドビュー編集（手動）

  1. リスト画面で「グリッドビューで編集」をクリック
  2. Excelからコピー＆ペーストで一括入力
  3. 「終了」で保存

方法B: 永賢組サイトのCSVエクスポート → PnP.PowerShellで自動投入（推奨）

永賢組サイト(constructioninfo)の各リストで「Excelにエクスポート」または ExportToCSV を実行すると、CSVには ListSchema={...}（先頭行）+ データ行 の両方が含まれます。Pythonでデータ部を抽出して PnP.PowerShell に流せば、スキーマ作成と同時にデータ投入も自動化できます。

仕組み:

  1. CSV をパースして {"items": [{"Title": "..."}, ...]} 形式の JSON に変換
  2. PnP.PowerShell で Add-PnPListItem -List "リスト名" -Values @{...} を各行に対して実行

サンプルコード（PowerShell）:

$schema = Get-Content "schema.json" -Raw | ConvertFrom-Json
foreach ($item in $schema.items) {
    $values = @{}
    foreach ($prop in $item.PSObject.Properties) {
        $values[$prop.Name] = $prop.Value
    }
    Add-PnPListItem -List "社員資格" -Values $values
}
💡 シンプルなマスタ系（社員資格・危険作業）に最適
Title列だけのシンプルなマスタリストはこの方法で1分で投入完了。Lookup を含む複雑なリスト（社員実績・OnsitePartnerRelation等）は、参照先リストの内部 ID 解決が必要なので、手動 or 拡張スクリプトで対応します。

👤 顔写真・個人情報の登録

作業員（職人さん）の個人情報・顔写真は、このステップでは投入しません。代わりに、Step6でNFCタグを準備した後、Step7のチラシで案内し、各作業員自身に登録してもらいます。
