建築現場の入退場管理システム

新企業（テナント）登録マニュアル v1.0

このマニュアルについて

顔認証ベースの建築現場入退場管理システムに、新しい企業（テナント）を追加するための初期登録手順をまとめたマニュアルです。

システムは マルチテナント構成 で、1つの基盤に複数の建設会社を収容できます。新規企業を迎える際は、契約締結 → SPO サイト構築 → 各種マスタ投入 → NFC タグ準備 → テスト → 本番運用 の流れで進めます。

📌 想定読者
クラウドパワー社 システム管理者・導入担当者
⚠️ 前提条件
  • Microsoft 365 テナント（nagaken.sharepoint.com）の管理者権限
  • Azure SQL Database sqldb-nagaken へのアクセス権
  • face.gemba-ai.cloud アプリのデプロイ権限
  • NFC タグ発行ツール

📋 メニュー

各ステップをクリックして詳細手順を確認してください。順番通りに進めることを推奨します。

1事前ヒアリング
契約条件・利用規模・認証方式を確認し、契約内容を確定
2契約先マスタ登録SPO
顔認証利用会社管理サイトに新テナントを登録
3テナント専用SPOサイト構築SPO
新企業専用のSharePointサイトを作成し、必要なリストを準備
4マスタリスト作成SPO
8種類のSPOリストを必要な列定義で作成
5初期データ投入SPO
社員・協力業者・現場・施工体制・危険作業マスタなどを投入
6NFCタグ準備・現場紐付けGUI
物理タグの調達 + face.gemba-ai.cloud/onsitenfctag で現場と紐付け（手動）
7配布物・台紙準備配布
NFC台紙・案内チラシ・操作ガイドをテナント向けにカスタマイズ
8テスト・受入確認UAT
テスト現場で全機能を疎通確認、ユーザー受入テスト実施
9本番運用開始CS
本番現場での運用開始、監視・サポート体制の確立

🗺 全体フロー

[Step1] 事前ヒアリング ↓ [Step2] 契約先マスタ登録（顔認証利用会社管理サイト） ↓ [Step3] テナント専用SPOサイト構築 ↓ [Step4] マスタリスト作成（8種類） ↓ [Step5] 初期データ投入 ─┬─ 社員マスタ ├─ 協力業者マスタ ├─ 現場マスタ ├─ 現場協力業者紐付け（施工体制ツリー） ├─ 社員資格マスタ／社員実績 └─ 危険作業マスタ／危険回避策マスタ ↓ [Step6] NFCタグ準備・現場紐付け（face.gemba-ai.cloud/onsitenfctag で手動登録） ↓ [Step7] 配布物・台紙準備（NFC台紙・チラシ・操作ガイド） ↓ [Step8] テスト・受入確認（テスト現場で疎通） ↓ [Step9] 本番運用開始
