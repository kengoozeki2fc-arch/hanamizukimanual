🗃️ Cosmos DB — Clients テーブルに新組織を追加

リソース名：child-camera（Azure Cosmos DB アカウント）／ コンテナ：Clients

フェーズ1：Clients コンテナに組織レコードを追加する

  1. 1
    Azure Portal にサインインし、Cosmos DB「child-camera」を開く
  2. 2
    左ペインから「Data Explorer」を選択
  3. 3
    コンテナ「Clients」を選択
  4. 4
    上部ツールバーの「New Item」をクリック

▲ Data Explorer → Clients.Items を開き「New Item」を押す

  1. 5
    右ペインに以下のJSONテンプレートを貼り付け、園情報を書き換える

    id と ClientID は 既存最大値＋1 を採番（既存が 1 なら 2）。PersonGroupId / StaffGroupId は 連番ルール（下記）で採番。

{
    "id": "2",
    "ClientID": 2,
    "OrganizationName": "ORGANIZATION NAME",
    "ClientName": "CLIENT NAME",
    "ClientKanaName": "KANA NAME",
    "ClientAddress": "ADDRESS",
    "ClientPhoneNo": "MOBILE NUMBER",
    "ClientPrefecture": "Japan",
    "Status": true,
    "CreatedDate": "2025-09-12T10:00:00Z",
    "UpdateDate":  "2025-09-12T10:30:00Z",
    "PersonGroupId": "hana-mizuki-g004",
    "StaffGroupId":  "hana-mizuki-s004"
}

▲ 右ペインへJSONを貼付し、上部ツールバーの「Save」を押す

  1. 6
    「Save item」をクリックしてレコードを保存

PersonGroupId / StaffGroupId の採番ルール

🔢
連番で増やすこと。Face API 内のディレクトリ名として使われるため、既存の最大番号＋1 を割り当てます。Blob Storage のフォルダー作成にも同じ値が使われます。
項目既存（例）新規割当（例）
PersonGroupIdhana-mizuki-g005hana-mizuki-g006
StaffGroupIdhana-mizuki-s005hana-mizuki-s006
⚠️
Face API ディレクトリは Azure Portal から作成する画面がありません。PersonGroupId / StaffGroupId を変更・追加した場合は 必ず開発担当（クラウドパワー）に連絡してください。こちらでディレクトリを作成します。

📱 Power Apps Solution のエクスポート・インポート

Power Apps の管理画面は Solution（ソリューション）単位で他環境へ移送します。開発元環境から ZIP エクスポートし、クライアント環境へインポートします。

フェーズ1：Power App を Solution に追加する

  1. 1
    https://make.powerapps.com にアクセス
  2. 2
    対象の環境を選択する
  3. 3
    左メニュー「Solutions（ソリューション）」へ移動
  4. 4
    「New Solution（新しいソリューション）」をクリック

    名前（例：ChildCameraSolution）／発行元／バージョン を入力

  5. 5
    作成した Solution を開く
  6. 6
    「既存を追加」→「App」→「Canvas App」をクリック
  7. 7
    対象の Power App を選択して追加する

フェーズ2：Solution を ZIP エクスポートする

  1. 1
    Solution 内で「Export」をクリック
  2. 2
    「Next」をクリック
  3. 3
    Managed / Unmanaged を選択し「Export」をクリック

    ・Managed：クライアント納品用（編集不可の封入形式）
    ・Unmanaged：開発用（編集可能）

  4. 4
    ZIP ファイルがダウンロードされる

フェーズ3：クライアント環境へ Solution をインポート

  1. 1
    クライアント環境（Power Platform テナント）にログイン

    https://make.powerapps.com

  2. 2
    環境セレクタで正しい環境を選択
  3. 3
    左メニュー「Solutions」へ移動
  4. 4
    「Import Solution」をクリック
  5. 5
    エクスポートした ZIP をアップロード
  6. 6
    「次へ」→「インポート」をクリック

フェーズ4：アプリケーションを確認する

  1. 1
    インポート後、Solution を開く
  2. 2
    以下を確認する
    • アプリが利用可能であること
    • 接続設定が正常に動作していること
  3. 3
    実機アクセスして基本動作をテストする
📝
簡易フロー： 開発元環境（Solution 作成 → ZIP エクスポート） → 本番環境（ZIP インポート → アプリ実行）
❗
重要事項
  • クライアント納品時は Managed Solution を使用してください。
  • インポート後、以下の再設定が必要な場合があります：
    API 接続 環境変数 権限設定
  • インポート後は必ず動作確認を実施してください。

🔑 クライアントID の変更（8つのコネクタ）

Cosmos DB の Clients コンテナで新規採番した ClientID（例：2）を、Power Apps 側の 8つのコネクタ呼び出しに反映します。 初期値はすべて 1（はなみずき）になっているので、新テナント用に書き換えてください。

ℹ️
最初のパラメータが ClientID を表します。書き換え後は アプリを保存し、公開 してください。

🆘
困ったときは

よくあるトラブル

  • Solution インポート時にエラー → API 接続・環境変数が未設定の可能性。Solutions → 当該 Solution → Connection References で再設定
  • アプリは開くが一覧が空 → ClientID の書換え漏れ。8つのコネクタをすべて確認
  • 顔登録で「登録に失敗」 → Face API ディレクトリが未作成。PersonGroupId / StaffGroupId を開発担当に連絡
  • 笑顔スコアダウンロードでメールが届かない → getEmailList / Email_Registration の ClientID を確認
  • 上記で解決しない場合は開発担当（クラウドパワー）に連絡
